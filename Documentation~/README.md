# Islands — Governed Documentation Root

Status: Active docs root for the migrated Islands package documentation.  
Scope: Navigation and role clarification for the governed `Documentation~/` tree.

## Purpose
This folder is the governed documentation root for the Islands Unity package.
It is not a public marketing surface and it is not a freeform dump of notes.
Its purpose is to make authority explicit and keep current truth separate from historical material.

## Read this first
1. `SSoT_INDEX.md`
2. `CURRENT_STATE.md`
3. the relevant file in `systems/`
4. the relevant governed file in `reference/`
5. the active or archived roadmap in `planning/`

## Authority model
- `systems/` contains the primary subsystem authorities that have earned promotion.
- `planning/active/` contains planning only. Planning is not implementation truth.
- `planning/archive/` contains superseded or absorbed roadmap history preserved for traceability.
- `reference/` contains governed support/reference material.
- `research/` contains ideas and investigation material.
- `archive/` contains historical material.
- `governance/` contains governance-specific support material when needed.

## Important warnings
- Files under the frozen snapshot or legacy `wip/` paths are migration input, not automatic authority.
- Old filenames containing `SSoT`, `Roadmap`, `Snapshot`, or similar labels do not automatically earn authority.
- Runtime-local markdown is not authoritative by location alone.
- When a concept changes, update its primary home first.
